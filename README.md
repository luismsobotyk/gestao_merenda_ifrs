## Sobre

O **SISGEM** - Sistema de Gestão da Merenda é um sistema web desenvolvido para simplificar, organizar e analisar a distribuição e o consumo da merenda escolar no IFRS. Focado na usabilidade e na análise de dados, o sistema permite o controle diário de retiradas, gestão de contratos, controle de discentes e geração de relatórios visuais. Esse projeto foi desenvolvido como parte do Trabalho de Conclusão do Curso Tecnólogo em Sistemas para Internet do IFRS - Campus Porto Alegre.

## ✨ Funcionalidades

* **Autenticação Segura:** Integração com o domínio institucional (LDAP/Active Directory) para início de sessão único (Single Sign-On) e gestão de permissões.
* **Gestão de Discentes e Cursos:** Controle de alunos autorizados a retirar a merenda e os seus respetivos cursos.
* **Controle de Contratos:** Registo e acompanhamento da vigência dos contratos de fornecimento de alimentos com gerência de empenhos e pedidos.
* **Cardápio Digital:** Interface interativa e responsiva para que os alunos possam consultar o menu do dia e da semana.
* **Registo de Retiradas:** Controle diário para registar a retirada da merenda pelos alunos.
* **Análise de Dados Avançada:** Dashboards completos para avaliar:
    * Aceitabilidade e tipos de merenda consumidos.
    * Distribuição e pico de consumo por dia da semana.
    * Consumo segmentado por turmas e cursos.
* **Avaliação de Usabilidade (SUS):** Formulário integrado (System Usability Scale) para medir a satisfação e a eficiência do sistema segundo os utilizadores.

## ⚙️ Pré-requisitos

Para executar este projeto no seu ambiente local, necessita de:

* PHP >= 8.2
* Composer (Gestor de pacotes PHP)
* Node.js e NPM (Para compilar os ficheiros de front-end)
* Servidor de Base de Dados (MySQL / MariaDB ou PostgreSQL)
* Extensão LDAP ativada no PHP (`php_ldap`)

## 🚀 Instalação e Configuração

Siga o passo a passo abaixo para configurar o SISGEM localmente:

**1. Clonar o repositório**

```bash
git clone https://github.com/luismsobotyk/gestao_merenda_ifrs
cd gestao_merenda_ifrs
```

**2. Instalar as dependências do PHP**

```bash
composer install
```

**3. Instalar as dependências do Frontend e compilar os assets**

```bash
npm install
npm run build
```

**4. Configurar o ambiente**

* Crie uma cópia do ficheiro de configuração padrão:

```bash
cp .env.example .env
```

* Abra o ficheiro `.env` e configure as credenciais da sua base de dados:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=sisgem_db
DB_USERNAME=root
DB_PASSWORD=
```

* Configure as credenciais do LDAP no ficheiro `.env` (se aplicável ao seu ambiente de testes).

**5. Gerar a chave da aplicação**

```bash
php artisan key:generate
```

**6. Executar as Migrations (Criação das Tabelas na Base de Dados)**

```bash
php artisan migrate
```

*(Opcional) Se pretender popular a base de dados com dados de teste predefinidos, pode executar: `php artisan migrate --seed`*

**7. Iniciar o Servidor Local**

```bash
php artisan serve
```

Agora pode aceder ao sistema através do seu navegador em: `http://localhost:8000`

## 📂 Estrutura do Sistema

O layout e os painéis administrativos (Dashboards) do sistema foram desenvolvidos priorizando a identidade visual institucional, utilizando uma paleta de tons verdes (`#1a8654` e `#1c6d46`) para transmitir segurança, equilíbrio e saúde, mantendo um design adaptável e responsivo tanto para computadores de secretária (desktops) como para dispositivos móveis (telemóveis e tablets).

## 📊 Dashboards e Gráficos

O módulo de "Análise de Dados" faz uso extensivo do **Apache ECharts** para apresentar métricas visuais claras sobre o fluxo e o consumo no refeitório.

Exemplos de Gráficos Implementados:

* **Distribuição Semanal (Gráfico de Setores/Doughnut):** Ilustra quais os dias da semana que registam maior fluxo de alunos.
* **Aceitabilidade (Gráfico de Barras):** Mede o índice de aceitação das diferentes opções integradas no cardápio.
* **Retiradas por Turma (Gráfico de Colunas):** Permite cruzar dados de consumo com as respetivas turmas e cursos dos alunos.
